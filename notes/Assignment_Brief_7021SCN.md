# Assignment Task

The tasks below cover module learning outcomes (1 to 5).

1.  **Tasks Overview**

This is an individual tasks. You are required to submit a portfolio of
practical tasks in secure software development, code analysis, and
compliance with security standards. The cybersecurity industry is full
of vendors selling expensive automated solutions that fail to fix
fundamental design flaws. This assignment requires you to prove that you
actually understand the underlying vulnerabilities, how to find them,
and how to fix them using standard open-source tools.

Your portfolio must be held as a single repository containing your code
and full documentation.

**2. Portfolio Tasks**

Your repository must contain the following distinct sections. These
reflect the practical work completed during your weekly lab sessions.

-   **Task A: Vulnerability Discovery and Remediation**\
    Provide a small, custom-written C or C++ programme that contains a
    deliberate memory corruption flaw, such as a buffer overflow.
    Document your method for exploiting it. Finally, provide the
    corrected code using secure coding practices.

-   **Task B: Threat Modelling and SDLC**\
    Include a threat model for a hypothetical web application. You
    should use the STRIDE methodology to identify potential threats and
    propose specific, secure requirements to mitigate them.

-   **Task C: Automated Security Testing**\
    Submit the configuration files for a continuous
    integration/continuous deployment (CI/CD) pipeline that
    automatically runs static analysis on your code. You must include
    the output of a scan, using any scanning tools like Semgrep or
    Sonarqube, along with a brief explanation of how you triaged the
    inevitable false positives.

-   **Task D: Dynamic Analysis and Exploit Development**\
    Document a web application penetration test against a local,
    vulnerable target. Include screenshots or logs from any tools that
    you use, such as OWASP ZAP or Burp Suite, proving your understanding
    of flaws like SQL injection or cross-site scripting (XSS).

-   **Task E: Compliance and Supply Chain**\
    Generate a Software Bill of Materials (SBOM) for a containerised
    application. Briefly explain how this artifact helps maintain
    compliance with frameworks like the NIST Secure Software Development
    Framework (SSDF).

**3. The Report**

Alongside the technical components, your portfolio will include a short
report (500 words) on the contents of the repo. This is not a
descriptive list of what is in the folder. It must be a critical
reflection on the security posture of your work. Discuss the limitations
of the tools you used and the real-world challenges of integrating
security into the software development lifecycle (SDLC).

# Assessed Module Learning Outcomes

The Learning Outcomes for this module align to the [marking
criteria](#assessment-marking-criteria) which can be found at the end of
this brief. Ensure you understand the marking criteria to ensure
successful achievement of the assessment task. The following module
learning outcomes (MLOs) are assessed in this task:

1.  Critically identify and categorize software security threats and
    vulnerabilities,evaluating their potential impacts and mitigation
    strategies.

2.  Integrate secure coding practices into the software development
    lifecycle(SDLC)and design robust systems that address common
    vulnerabilities.

3.  Apply and critically assess software security testing strategies,
    including static and dynamic analysis,to identify and mitigate
    security flaws.

4.  4\. Evaluate and implement compliance with software security
    standards such as the Secure Software Development Framework (SSDF),
    demonstrating an understanding of their practical applications.

5.  Analyse and critique current and emerging software security
    challenges, proposing innovative and sustainable solutions.
